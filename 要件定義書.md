# Roopie(Chromiumベース独自ブラウザ) 要件定義書 v0.2

作成日: 2026-07-13
作成者: (ユーザー) × Claude
プロジェクト名: **Roopie**

---

## 1. プロジェクトコンセプト

Chromium(Web標準エンジン)をベースに、**作業効率化機能**と**用途ごとに完全に切り替えられるプロファイル機能**を軸にした、モダンでスマートなデザインの独自ブラウザを開発する。

開発者はプログラミング未経験。実装は Claude Code に一任し、ユーザーは要件定義・意思決定・レビューを担当する。

---

## 2. 開発方針・技術スタックの提案

### 2.1 ベース技術
「Chromiumベース」の実現方法には複数の選択肢があるが、以下の理由から **Electron** を採用することを提案する。

| 方式 | 難易度 | Claude Codeとの相性 | 拡張機能対応 |
|---|---|---|---|
| 本家Chromiumソース改造(C++) | 非常に高い(数ヶ月〜) | 低い | ◎(完全) |
| **Electron(JS/TypeScript)** | **中** | **高い** | △(部分対応・要工夫) |
| CEF(C++寄り) | 高い | 低い | ◎ |
| 既存ブラウザの拡張機能 | 低い | 高い | - |

**⚠️ 重要な技術的懸念事項(要合意)**
Chrome Web Storeの拡張機能を「必須」とのご要望だが、ElectronはChromeそのものではないため、拡張機能(特にuBlock Originなど広告ブロック系が使うMV3のdeclarativeNetRequest等)を**ネイティブには完全サポートしていない**。`electron-chrome-web-store` 等のコミュニティライブラリで多くの拡張機能を動かせるが、100%の互換性は保証されない。
→ 開発初期(フェーズ2)で実際に主要拡張機能(uBlock Origin等)を動かして検証し、致命的な問題があれば方針を再検討する「検証マイルストーン」を設ける。

### 2.2 将来のクロスプラットフォーム展望
Electronは元々Win/Mac/Linuxに対応しているため、将来の展開は技術的に無理なく可能。スマホ対応(iOS/Android)は別エンジンが必要なため、完全に別プロジェクト(将来フェーズ)として扱う。

## 3. 対象環境

- **フェーズ1〜: Windows** のみ対応
- 将来的に: Mac → Linux → (別プロジェクトとして)スマホアプリ

## 4. 機能要件

### 4.1 プロファイル機能(最重要機能)
用途ごとに「Web上の人格」を切り替えられる機能。プロファイルごとに以下の項目を**個別に「共有する/しない」を設定可能**にする。

- Cookie・ログインセッション
- 保存パスワード
- デフォルトログインするGoogleアカウント等
- スタートページの見た目・レイアウト
- ショートカットキー設定
- マウスジェスチャー設定
- ブックマーク
- 使用するプロキシ/VPN設定
- 拡張機能の有効/無効
- テーマ(配色・アイコン)

→ 「プロファイル管理画面」で、上記の各項目ごとにトグルで共有可否を切り替えられるUIを想定。

### 4.2 サイドパネル
Edge/Vivaldiのサイドパネルをベースに、以下を統合した高機能パネルとして拡張:

- ブックマーク/履歴へのクイックアクセス
- ページを見ながら使えるメモ機能
- AIチャット機能(ページ内容を参照した質問など)
- タブ連携機能(リードリスト、あとで読む等)
- **Webパネル機能**: Instagram, X(Twitter), Messenger, Gmailなど任意のWebサイトをサイドパネルに常駐表示できる機能(Opera/Vivaldiの「ウェブパネル」に相当)。メインのブラウジングをしながら、SNSの閲覧やメッセージ・メールの送受信を並行して行える
  - 複数サイトを登録し、パネル内でアイコン切り替えできるようにする
  - ログイン状態の保持が必要なため、4.1のプロファイル機能(Cookie管理)と連動して動作させる
- **カレンダー・TODO機能**: Googleカレンダー等の外部サービスと連携したスケジュール確認・TODO管理をサイドパネル上で行える機能(閲覧しながら予定確認・タスク追加ができる)

### 4.3 メディアプレイヤー
Appleライク(macOSのミニプレイヤーのような)洗練されたデザインのメディアプレイヤー。

- 画面の四隅いずれかに常駐表示できるフローティング型プレイヤー(動画/音楽サイト再生時に自動表示等を想定)
- サイドパネルへ格納することも可能(常駐か格納かをユーザーが切り替え)
- **ピクチャインピクチャ(PinP)対応**: 動画再生中に、ブラウザウィンドウの外にも独立した小窓として飛び出させられる(常時最前面表示、他アプリ作業中も視聴継続可能)。フローティングプレイヤー(四隅常駐)とPinP(ウィンドウ外に独立)は用途で使い分けられるようにする

### 4.4 画面分割機能
1つのウィンドウ内で最大2画面までの分割表示。

- 分割方向は縦(左右)・横(上下)を切り替え可能
- 2つの異なるタブ/ページを同時に表示しながら閲覧・作業ができる

### 4.5 マウスジェスチャー
Vivaldi/Edgeのようなブラウザ標準搭載のジェスチャー機能(右クリック+ドラッグで「戻る」「進む」「タブを閉じる」等)。設定はプロファイルごとに切り替え可能(4.1参照)。

### 4.6 拡張機能対応
Chrome Web Storeの既存拡張機能をインストール・利用可能にする(2.1の技術的懸念を参照)。

### 4.7 UIテーマカスタマイズ機能
ブラウザ全体のUI(タブバー・サイドパネル・設定画面など)の見た目を、ユーザーが高い自由度でカスタマイズできる機能。

- **テーマエディタ(GUI)**: フォント・カラー・背景画像をGUI上で自由に組み合わせて設定可能
- **上級者向けカスタムCSS機能**: 自分で直接CSSを記述し、UIに適用できる機能(Bonjourrの「カスタムCSS」機能や、Vivaldi/Stylusのようなイメージ)
- **プロファイル連動**: 4.1のプロファイル機能と連動し、プロファイルごとに異なるテーマ(フォント・配色・背景・カスタムCSS)を割り当てられる。プロファイルの「共有/独立」設定項目の一つとして「テーマ」を管理する

### 4.8 デザイン
Arc BrowserやZennのような、モダン・先進的・Appleライクな洗練されたUIを目指す。

**デザインシステム: Tailwind CSS + Preline UI を採用**
- 参考: [Preline UI](https://preline.co/)(Tailwind CSSベースの無料UIコンポーネントライブラリ)
- 特徴: ミニマルな余白設計、角丸カード/ボタン、ダークモード標準対応、情報設計重視の「機能的で美しい」SaaS的UI
- ElectronのUI部分(タブバー・サイドパネル・プロファイル切り替え画面等)はHTML/CSS/JSで構築するため、Tailwind CSS + Prelineのコンポーネントをそのまま部品として活用できる
- ダークモードは標準搭載する方針

**スタートページ(新規タブ)のデザイン: Bonjourrを参考**
- 参考: [Bonjourr](https://bonjourr.fr/)(iOS風ミニマル新規タブ拡張機能)
- 特徴: 時間帯で変化するフルスクリーン高画質背景写真、中央に大きなアナログ/デジタル時計、控えめな検索バー、アイコンのみのクイックリンク(ブックマーク)、装飾を削ぎ落とした「静かで美しい」世界観
- Preline UI(機能重視のダッシュボード的UI)とは役割を分け、**スタートページだけはBonjourr的な「引き算の美学」、それ以外の操作画面(サイドパネル・設定・プロファイル管理等)はPreline UI的な「機能的なSaaS UI」**という使い分けを基本方針とする


---

## 5. 非機能要件

- 動作の軽快さ(素のChromeと比べて極端に重くならないこと)
- セキュリティ: パスワード・Cookie等の機密情報は暗号化して保存
- 将来の機能追加・保守がしやすいコード構成(Claude Codeでの継続開発を前提)

## 6. 開発の進め方(フェーズロードマップ・提案)

まずは小さく動くものを作り、段階的に機能を足していく方針とする。

| フェーズ | 内容 | ゴール |
|---|---|---|
| **Phase 0** | 開発環境構築(Node.js, Claude Code連携, Git等) | Claude Codeで開発を始められる状態 |
| **Phase 1** | Electronで最小構成のブラウザ(URL入力・戻る進む・タブ)を作成 | 「ブラウザとして動く」ものが完成 |
| **Phase 2** | 拡張機能対応の検証(uBlock Origin等が動くか実験) | 技術的実現可否の確認(重要な分岐点) |
| **Phase 3** | プロファイル機能の実装(まずCookie/ブックマーク単位から) | 人格切り替えの基本形が完成 |
| **Phase 4** | マウスジェスチャー・サイドパネル実装 | 効率化機能が使える状態 |
| **Phase 5** | デザインの作り込み(Arc/Zenn風UI適用) | 見た目の完成度を上げる |
| **Phase 6** | 細部の作り込み・不具合修正・将来のMac/Linux対応検討 | 実用レベルへ |
