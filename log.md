### 動く機能(一覧)

- **ブラウザ基本**: タブ(Ctrl+T/W/Tab、中クリックで閉じる、Ctrl+1〜9)、アドレスバー(URL判定/Google検索)、戻る・進む・再読み込み、ズーム(Ctrl+ +/−/0)、ページ内検索(Ctrl+F)、右クリックメニュー、印刷(Ctrl+P)、全画面、マウスの戻る/進むボタン
- **データ**: ブックマーク(Ctrl+D、バー Ctrl+Shift+B、管理 Ctrl+Shift+O)、履歴(Ctrl+H)、ダウンロード(Ctrl+J)。**管理画面は通常のブックマークとスタート画面のショートカットを1つのフォルダツリーで表示**(フォルダ折りたたみ・名前変更・削除・「移動」でブックマーク⇔スタート画面の各ページを行き来・検索は所在パス付きで横断。startルート自体は削除・改名不可)。サイドパネルのブックマークは「ブックマークを追加」ボタンからURL+名前のモーダルで追加可。検証: `npx electron scripts/test-bookmarks-manager.js`
- **プロファイル(Edge挙動)**: タブバー左端のワークスペース風ピルからプルダウンで切り替え。**切り替え=そのプロファイルの新しいウィンドウを開く**(既存ウィンドウは元のプロファイルのまま。複数プロファイルを同時に使える)。ウィンドウの無いプロファイルを開くと前回のタブ構成を復元(最後のウィンドウを閉じるときに保存)。プロファイル削除でそのウィンドウも閉じる。設定画面(Ctrl+,)で追加・名前変更・削除、共有トグル(ブックマーク/履歴/設定/パスワード/自動入力など。データ束はウィンドウのプロファイル単位)。設定画面のプロファイルカードから切り替えずにテーマカラー変更可。検証: `npx electron scripts/test-multi-profile.js`
- **タブバーの位置**: 上部(既定)/左側(縦)をツールバーのアイコンで即切り替え(グローバル設定)。縦タブ時は上部に専用ストリップを置かず、ツールバーがタイトルバー(ドラッグ領域)を兼ねる=デッドスペースなし。切替ボタン+ワークスペースピルはレール最上部の1行に入る
- **タブの切り離し**: タブをタブバーの外へドラッグ&ドロップすると新しいウィンドウとして開く(最後の1枚は不可)
- **Tor接続(プロファイル別)**: 設定画面のプロファイルカードの「Torで接続」トグルで、そのプロファイルの通信をTorネットワーク経由にできる。プロファイル(=セッション)ごとに独立(あるプロファイルはTor、別のプロファイルは直接接続を同時に使える)。使用中はワークスペースピルに🧅インジケーター。**Tor本体(tor.exe)を同梱しているので追加インストールは不要**(`scripts/fetch-tor.js` が公式Expert BundleからSHA256照合付きで取得 → `vendor/tor` → extraResourcesで同梱。**CIがリリースのたびに最新の安定版を取り直すのでRoopieの更新でTorも新しくなる**)。探索順は ①`%APPDATA%/Roopie/tor/tor.exe`(ユーザーの差し替え用) ②同梱 ③インストール済みTor Browser。なお起動時にまず9050/9150を見て、既に動いているTorがあればそれを使う。同梱バージョンは設定画面のTor状態表示に出る。ブリッジ(pluggable transports)とgeoipは同梱していない。検証: `npx electron scripts/test-tor.js`(実際にBootstrapped 100%まで確認。30〜90秒)
- **プロファイルアイコン**: 設定画面でアバターをクリックすると選択パネル(既定の絵文字24種+自由入力+画像アップロード+文字に戻す)。画像アップロード時はドラッグ&ズームのGUIクロップ(円形プレビュー)。ツールバー・プロファイルメニュー・設定画面すべてに反映
- **Googleアカウント**: ブラウザ全体に登録し、プロファイルごとに有効化+プライマリ選択。**ログイン中のアカウントを自動検出して登録・有効化**(設定画面を開いたとき、およびタブがgoogle.com系ドメインへ遷移したときの2箇所でチェック。手動登録も引き続き可能)
- **マウスジェスチャー**: 右クリック+ドラッグ(軌跡+アクション名を表示)。既定: ←戻る / →進む / ↓新しいタブ / ↓→タブを閉じる / ↑↓再読み込み。設定画面のGUIで自由に追加・変更・削除でき、共有トグルで全プロファイル共通にもできる

### 初回起動 / アップデート後の案内

- **初回起動**: `roopie://welcome` を最初のタブに開く。6ステップ(ようこそ → できること → 見た目 → 検索とプライバシー → お住まいの場所 → 完了)で、アクセントカラー・タブバー位置・検索エンジン・広告ブロック・天気の場所をその場で決められる(選んだ瞬間に反映)。「スキップ」でいつでも抜けられる
- **アップデート後**: `roopie://whatsnew` を最初のタブに開き、`src/renderer/pages/release-notes.json` の変更点を表示する
- **出し分けはアプリのバージョンではなく release-notes.json の先頭 version(0.1 / 0.2 …)で行う**。masterへのpushごとにビルド番号(0.1.<run>)が上がるため、バージョン一致で判定すると1コミットごとにポップアップが出てしまう。**VERSION_BASE を上げるときに release-notes.json へ1件足す**と、全ユーザーが起動時に1度だけ変更点を見る
- 状態は `userData/app-state.json`(`introDone` / `seenNotes` / `installedVersion`)。開発中は `ROOPIE_ONBOARDING=welcome|whatsnew|off` で強制表示できる
- 設定画面の「Roopieについて」からいつでも再表示できる(変更履歴は `roopie://whatsnew?all` で全件)。バージョン表示・「更新を確認」・GitHubを開くも同じ場所
- 検証: `npx electron scripts/test-onboarding.js`

### リリース(自動)

masterへpushすると `.github/workflows/release.yml` がwindows-latestでビルドし、GitHub Releasesへ公開する。インストール済みのRoopieは起動時+15分ごとにReleasesを確認し、新しければ裏でダウンロードして再起動を促す。**手動で `npm run release` を叩く必要はない**(ローカル確認だけしたいときは `npm run dist`)。

- バージョン: `0.1.<Actionsの実行番号>` を**ビルド時にだけ**適用。package.jsonは0.1.0のまま=CIがmasterへpushし返さないので無限ループしない。メジャー/マイナーを上げるときはワークフローの `VERSION_BASE` を変える
- **このワークフローファイルはリネーム・移動しない**(実行番号が1に戻り、バージョンが逆行して自動更新が止まる)
- `publish.releaseType: "release"` は必須(既定の `draft` だとelectron-updaterから見えない)
- 公開状態のリリースはタグが先に必要なため、ビルド対象のコミットにタグを打ってから公開する。またelectron-builderは成果物を並列アップロードして同じタグのリリースを二重に作ることがあるため、**空のリリースを先に作ってからビルドする**
- 成果物は1回約100MBなので古いリリースは直近10件だけ残す(手動作成のv0.1.0もいずれ消える)。`**.md` だけの変更ではビルドしない
- 反映までの目安: push → ビルド約2分 → アプリが最大15分後に検知 → 再起動で適用
- インストーラーはNSISのアシスト形式(oneClick:false / ユーザー単位)。インストール先の変更・デスクトップショートカット・完了後の起動に対応。左側とヘッダーの画像は `npx electron scripts/gen-installer-art.js` で生成する(build/*.bmp。NSISは24bit BMPのみ対応のため自前で組み立てている)
- **アンインストール時に「ユーザーデータを残すか」を尋ねる**(`build/installer.nsh` の `customUnInstall`)。既定は「残す」で、いいえを選んだときだけ `%APPDATA%\Roopie` を削除する。**自動アップデート(--updated)やサイレント実行では尋ねず必ず残す**(これがないと更新のたびにデータが消える)

### 既知の制約・未対応

- **uBlock Originなどのブロッキング型拡張は動かない**(Electronの制約。内蔵広告ブロックで代替済み。詳細は下の Phase 2 検証記録)
- 画面分割は非アクティブペイン(相方タブ)側でツールバー(戻る/進む/ズーム/アドレスバー)を操作できない。ページ内でのクリック・スクロール・右クリックメニューは相方ペインでも正しく動く
- シークレットウィンドウでは拡張機能が動かない(Electronの制約: 非永続セッションは非対応)
- 起動ログの `blink.mojom.WidgetHost` エラーはChromiumの無害なノイズ(対応不要)
